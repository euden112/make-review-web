# Layer 3 분석 — 점수 정합성 (score alignment)

**질문**: 파이프라인이 내는 결정론적 점수(총점·항목별)가 실제 리뷰 데이터의 추천 분포와 일치하는가?
이는 단순 베이스라인이 **구조적으로 만들지 못하는** 산출물이라, 정합함을 보이면 설계 가치를 직접 증명한다.

**방법**: 정합도 = `1 − |실제추천율 − 점수정규화|` (0~1).
- 요약 점수: 라이브 API `/summary`(실배포 산출물) — sentiment_score(0~100)·aspect score(0~10).
- 원본: DB `external_reviews`(steam, is_recommended) + review_categories_json(한글 카테고리 = 모듈 CATEGORIES와 동일 어휘).
- 표본: 코어셋 20개. 원자료 `alignment_results.csv`

## 1. 결과 — 두 축 각각 (독립 보고)

| 축 | 평균 정합도 | 의미 |
|---|---|---|
| **총점 정합성** | **0.909** | 요약 총점 ↔ 원본 추천율 |
| **카테고리 정합성** | **0.842** (coverage 1.00) | 항목 점수 ↔ 항목별 추천율 |
| (보조) macro | 0.876 | 단일 점수 뭉뚱그림 지양 |

- **coverage 1.00**: 코어셋 20개 모두 5개 카테고리에 데이터가 있어 임의값 주입 없이 전 항목 평가됨.
- 총점은 파이프라인이 추천율 앵커에 정합하도록 설계됐으니 높게 나오는 게 정상.
  더 독립적인 검증은 **카테고리 축** — 항목 점수는 단순 앵커가 아니라 evidence에서 산출되는데도
  항목별 추천율과 0.842로 정합 → 결정론 점수 산출이 실데이터를 충실히 반영함.

## 2. 베이스라인 대비 — 이 축은 설계 고유

단순 단일프롬프트 베이스라인은 **결정론적 항목 점수(0~10)를 만들지 못한다** → 이 축에선 비교 자체가
성립하지 않는다.

## 3. 지표 신뢰도 — 실데이터 검증

`validate()` 4종(sanity·negative control·random·monotonic)을 합성이 아닌 **코어셋 실리뷰**로 재실행:
- pooled 코어셋 실리뷰(5,855건): **✅ 4종 전체 통과**.
- 게임별: **18/20 통과**.

### 2건 실패
- 실패 게임: **Battlefield 2042**(추천율 0.46), **The Outer Worlds**(0.59) — 둘 다 **호불호 ~50%**.
- 원인: validate의 negative-control 하위검사는 "최대로 어긋난 점수 → 정합도 < 0.4"를 요구하는데,
  추천율이 ~0.5인 게임은 가장 어긋난 점수(0★ 또는 5★)조차 `|0.5−0.5|=0.5`만큼만 벗어나 정합도가
  ~0.5 밑으로 못 내려간다. 즉 **절대 임계 0.4가 균형분포 게임에선 구조적으로 도달 불가**.
- sanity·random·monotonic 세 검사는 이 두 게임에서도 통과 → **지표의 판별력 자체는 유효**.


### 4. 저정합의 주원인은 (staleness)

The Outer Worlds·Battlefield 2042를 DB로 대조한 결과, 이 저정합은 점수 산출 결함이 아니라
**"크롤링은 계속 / 재요약은 안 됨"으로 생긴 시점 차**임이 확인됐다.

| 게임 | 요약일 | cursor(요약이 본 마지막 id) | 현재 max id | 커서 이후 신규 Steam 리뷰 | 추천율(요약 당시→현재) | sentiment(score_norm) | 총점 정합 |
|---|---|---|---|---|---|---|---|
| Outer Worlds | 06-06 | 26483 | 39580 | 36 | **82.4% → 59%** | 82 (0.82) | 0.77 |
| Battlefield 2042 | 06-06 | 26383 | 53445 | 50 | 47.1% → 46.3% | 44 (0.44) | **0.98** |

- 요약 점수는 **둘 다 요약 당시 추천율엔 정확히 맞았다**(82.4%→82, 47.1%→44). 그런데 그 뒤로도
  Steam 크롤이 매일(~17:08 UTC) 신규 리뷰를 쌓는데 요약은 06-06에 멈춰 있어(cursor ≪ 현재 max id),
  pos_rate(현재 추천율)만 갱신되고 score_norm(요약 점수)은 옛값에 고정됐다.
- **대조가 결정적**: 추천율이 거의 안 움직인 Battlefield는 옛 점수로도 0.98로 정합, 추천율이 크게
  움직인 Outer Worlds(82→59)만 +0.23 어긋났다 → 어긋남의 크기 = 추천율 이동폭. **점수가 틀린 게
  아니라 비교 대상이 옛 점수**라는 뜻.
- **재요약 시 일치 예상**: 총점은 추천율에 앵커되도록 설계됐으므로, 현 DB로 재요약하면 새 sentiment가
  현재 추천율(≈59)을 따라가 정합도가 ~1.0으로 회복될 것으로 예상된다(±8 LLM delta 한도라 완전 일치는
  아닐 수 있음). **단 이는 실측이 아닌 설계상 예상** — 재요약을 시도했으나 Groq 일일 quota 소진으로
  reduce가 실패해(`reduce error: quota`) 현재는 검증하지 못했다. quota 리셋 후 75·78 force 재요약
  → alignment 재실행으로 확인 가능.

> 판별력이 검증됐다
