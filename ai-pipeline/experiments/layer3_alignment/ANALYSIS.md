# Layer 3 분석 — 점수 정합성 (score alignment)

**질문**: 파이프라인이 내는 결정론적 점수(총점·항목별)가 실제 리뷰 데이터의 추천 분포와 일치하는가?
이는 단순 베이스라인이 **구조적으로 만들지 못하는** 산출물이라, 정합함을 보이면 설계 가치를 직접 증명한다.

**방법**: LLM 없이 순수 산술(stdlib·결정론). 정합도 = `1 − |실제추천율 − 점수정규화|` (0~1).
- 요약 점수: 라이브 API `/summary`(실배포 산출물) — sentiment_score(0~100)·aspect score(0~10).
- 원본: DB `external_reviews`(steam, is_recommended) + review_categories_json(한글 카테고리 = 모듈 CATEGORIES와 동일 어휘).
- 표본: 코어셋 20개(시드 20260607, 1·2층과 동일). 원자료 `alignment_results.csv`, 자동 리포트 `alignment_report.md`.

## 1. 결과 — 두 축 각각 (독립 보고)

| 축 | 평균 정합도 | 의미 |
|---|---|---|
| **총점 정합성** | **0.909** | 요약 총점 ↔ 원본 추천율 |
| **카테고리 정합성** | **0.842** (coverage 1.00) | 항목 점수 ↔ 항목별 추천율 |
| (보조) macro | 0.876 | 단일 점수 뭉뚱그림 지양 |

- **coverage 1.00**: 코어셋 20개 모두 5개 카테고리에 데이터가 있어 임의값 주입 없이 전 항목 평가됨.
- 총점은 파이프라인이 추천율 앵커에 정합하도록 설계됐으니 높게 나오는 게 정상(설계 작동 확인).
  더 독립적인 검증은 **카테고리 축** — 항목 점수는 단순 앵커가 아니라 evidence에서 산출되는데도
  항목별 추천율과 0.842로 정합 → 결정론 점수 산출이 실데이터를 충실히 반영함.

## 2. 베이스라인 대비 — 이 축은 설계 고유

단순 단일프롬프트 베이스라인은 **결정론적 항목 점수(0~10)를 만들지 못한다** → 이 축에선 비교 자체가
성립하지 않는다. (폐지된 ablation이 faithfulness 축에서 보이지 못한 "설계 가치"를, Layer 3가
베이스라인이 못 만드는 산출물의 데이터 정합성으로 정면 증명한다.)

## 3. 지표 신뢰도 — 실데이터 검증

`validate()` 4종(sanity·negative control·random·monotonic)을 합성이 아닌 **코어셋 실리뷰**로 재실행:
- pooled 코어셋 실리뷰(5,855건): **✅ 4종 전체 통과**.
- 게임별: **18/20 통과**.

### 2건 실패는 지표 결함이 아님 (정직 보고)
- 실패 게임: **Battlefield 2042**(추천율 0.46), **The Outer Worlds**(0.59) — 둘 다 **호불호 ~50%**.
- 원인: validate의 negative-control 하위검사는 "최대로 어긋난 점수 → 정합도 < 0.4"를 요구하는데,
  추천율이 ~0.5인 게임은 가장 어긋난 점수(0★ 또는 5★)조차 `|0.5−0.5|=0.5`만큼만 벗어나 정합도가
  ~0.5 밑으로 못 내려간다. 즉 **절대 임계 0.4가 균형분포 게임에선 구조적으로 도달 불가**.
- sanity·random·monotonic 세 검사는 이 두 게임에서도 통과 → **지표의 판별력 자체는 유효**.
  (핵심 지표 함수는 원본 그대로 두는 원칙이라 validate 임계는 수정하지 않고 한계로 명시.)

## 4. 저정합 게임 정성 검토 (점수 오류 vs 지표 오류)

- **Street Fighter 6 (total 0.676, 최저)**: 요약 총점 정규화 0.88 vs 실제 추천율 0.556 → **+0.32 과대평가**.
  요약이 SF6를 88% 긍정으로 봤으나 실제 추천은 55.6%. 카테고리 최적화도 0.53으로 낮음.
- **The Outer Worlds (0.77)**: 0.82 vs 0.59 (+0.23 과대), **Forza Horizon 5 (0.842)**: 0.89 vs 0.73 (+0.16 과대).
- **Overwatch 2 (cat 0.714 최저)**: 추천율 0.42 vs 총점 0.25 (−0.17 과소). 리뷰밤 게임 특성.
- **해석/주의**: 과대평가 군은 두 가능성 — ① 실제 점수 산출 결함(추천율 대비 총점이 높게 책정),
  또는 ② **라이브 요약 생성 시점의 리뷰 모집단 ≠ 현재 DB 크롤 모집단**(시점 차로 추천율이 이동).
  총점 축은 이 staleness에 민감하므로, 발표 시 "총점 정합은 시점 정합도 함께 본다"고 명시.
  카테고리 축은 같은 데이터 계보라 이 영향이 작다.

## 5. 가정 (방어 포인트)
- "추천율 p ↔ 별점 5p" **선형 매핑** 전제 — 투명·조정 가능(STAR_MAX/CATEGORY_MAX·매핑 교체).
  Steam 등급이 별점과 비선형일 수 있다는 반론엔 "선형 매핑 사용, 조정 가능"으로 대응.
- 데이터 없는 카테고리는 임의값 주입 없이 평균 제외 + coverage 보고(점수 부풀림 방지). 본 코어셋은 coverage 1.00.

## 6. 발표 한 줄 (§3 3층)
> 파이프라인의 결정론 점수는 총점 0.909·카테고리 0.842로 실제 리뷰 추천 분포와 정합한다(coverage 1.00).
> 이 점수는 단순 베이스라인이 구조적으로 만들 수 없는 산출물이며, 지표 자체는 실데이터 validate 4종으로
> 판별력이 검증됐다(균형분포 2게임은 negctrl 절대임계 한계로 일부 미발화 — 지표 결함 아님). 과대평가
> 사례(SF6 등)는 점수 calibration·시점 staleness 관점에서 정직하게 보고.
